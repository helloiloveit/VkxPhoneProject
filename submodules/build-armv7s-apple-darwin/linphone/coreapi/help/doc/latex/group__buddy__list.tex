\subsection{Detailed Description}
{\bfseries Buddies and buddy list} \par
Each buddy is represented by a \doxyref{Linphone\-Friend}{p.}{group__buddy__list_ga2f0508bab6d8ded431ce8b1a679a1522} object created by function \doxyref{linphone\-\_\-friend\-\_\-new()}{p.}{group__buddy__list_ga97cbc9a6c2668bde387ed83a665559c5}. Buddy configuration parameters like \doxyref{sip uri }{p.}{group__buddy__list_ga4b95bb01402026b8ffc7a7be9729af93} or \doxyref{status publication }{p.}{group__buddy__list_ga1bef7a3f4194c5a8084897af07dfc5fb} policy for this \doxyref{friend }{p.}{group__buddy__list_ga2f0508bab6d8ded431ce8b1a679a1522} are configurable for each buddy. \par
Here under a typical buddy creation\-: \par
 
\begin{DoxyCode}
LinphoneFriend* my\_friend=linphone_friend_new_with_addr(\textcolor{stringliteral}{"sip:joe@sip.linphone.org"}); \textcolor{comment}{/*creates friend
       object for buddy joe*/}
linphone_friend_enable_subscribes(my\_friend,TRUE); \textcolor{comment}{/*configure this friend to emit SUBSCRIBE message after
       being added to LinphoneCore*/}
linphone_friend_set_inc_subscribe_policy(my\_friend,LinphoneSPAccept); \textcolor{comment}{/* accept Incoming subscription
       request for this friend*/}
\end{DoxyCode}
 \doxyref{friends }{p.}{group__buddy__list_ga2f0508bab6d8ded431ce8b1a679a1522} status changes are reported by callback \doxyref{Linphone\-Core\-V\-Table.\-notify\-\_\-presence\-\_\-recv}{p.}{struct__LinphoneVTable_afedfc364c5b5193f10473cd47524400f} 
\begin{DoxyCode}
\textcolor{keyword}{static} \textcolor{keywordtype}{void} notify\_presence\_recv\_updated (\textcolor{keyword}{struct} \_LinphoneCore *lc,  
      LinphoneFriend *\textcolor{keyword}{friend}) \{
        \textcolor{keyword}{const} LinphoneAddress* friend\_address = linphone_friend_get_address(\textcolor{keyword}{friend});
        printf(\textcolor{stringliteral}{"New state state [%s] for user id [%s] \(\backslash\)n"}
                                ,linphone_online_status_to_string(
      linphone_friend_get_status(\textcolor{keyword}{friend}))
                                ,linphone_address_as_string (friend\_address));
\}
\end{DoxyCode}
 \par
Once created a buddy can be added to the buddy list using function \doxyref{linphone\-\_\-core\-\_\-add\-\_\-friend()}{p.}{group__buddy__list_gaf049534f1d2a76cbe4e62abed8e2c096} . Added friends will be notified about \doxyref{local status changes }{p.}{group__buddy__list_ga0a624a6e999108ee052b89e743698227} \par
 Any subsequente modifications to \doxyref{Linphone\-Friend}{p.}{group__buddy__list_ga2f0508bab6d8ded431ce8b1a679a1522} must be first started by a call to function \doxyref{linphone\-\_\-friend\-\_\-edit()}{p.}{group__buddy__list_ga4329c451eb96f1b5073990bb2212c52b} and validated by function \doxyref{linphone\-\_\-friend\-\_\-done()}{p.}{group__buddy__list_gae73b009008c6c6ec92ea3de71135d5f7} 
\begin{DoxyCode}
linphone_friend_edit(my\_friend); \textcolor{comment}{/* start editing friend */}
linphone_friend_enable_subscribes(my\_friend,FALSE); \textcolor{comment}{/*disable subscription for this friend*/}
linphone_friend_done(my\_friend); \textcolor{comment}{/*commit changes triggering an UNSUBSCRIBE message*/}
\end{DoxyCode}


{\bfseries  Publishing presence status } \par
Local presence status can be changed using function \doxyref{linphone\-\_\-core\-\_\-set\-\_\-presence\-\_\-model()}{p.}{group__buddy__list_ga9b0cdc2b8ac83ec6db8ca9502e9c2972} .New status is propagated to all friends \doxyref{previously added }{p.}{group__buddy__list_gaf049534f1d2a76cbe4e62abed8e2c096} to \doxyref{Linphone\-Core}{p.}{group__initializing_gaa5cf635b82dd338e7ee2dd3599d05f0a}.

{\bfseries Handling incoming subscription request} \par
 New incoming subscription requests are process according to \doxyref{the incoming subscription policy state }{p.}{group__buddy__list_ga1bef7a3f4194c5a8084897af07dfc5fb} for subscription initiated by \doxyref{members of the buddy list. }{p.}{group__buddy__list_gaf049534f1d2a76cbe4e62abed8e2c096} \par
 For incoming request comming from an unknown buddy, the call back \doxyref{Linphone\-Core\-V\-Table.\-new\-\_\-subscription\-\_\-request}{p.}{struct__LinphoneVTable_ae66dffa2d5e6b3be881552cab3024ec2} is invoked.
